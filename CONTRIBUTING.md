# Contributing to Forge

Thanks for wanting to help build Forge. This is a source-available project (see
[LICENSE](LICENSE)) — you're welcome to fork, modify, and contribute. The one thing the
license forbids is reselling Forge itself as a competing commercial product; using it (even
commercially) and contributing to it are fully encouraged.

## Getting set up

You need the standard Roblox Luau toolchain:

| Tool | Purpose | Install |
|------|---------|---------|
| [Rojo](https://rojo.space) | build/serve the plugin | `cargo install rojo` or `aftman add rojo-rbx/rojo` |
| [selene](https://kampfkarren.github.io/selene/) | linting | `cargo install selene` |
| [StyLua](https://github.com/JohnnyMorganz/StyLua) | formatting | `cargo install stylua` |
| [luau](https://luau.org) (CLI) | running the test suite | `brew install luau` / build from source |

## Build & run

```bash
rojo build plugin.project.json --output Forge.rbxmx   # build the plugin
# then in Studio: right-click Plugins → Insert from File → Forge.rbxmx
```

For live development: `rojo serve plugin.project.json` and connect via the Rojo Studio plugin.

The generation engine runs in Claude Code with the Roblox Studio MCP server connected — start
it with `/forge` (see [docs/ENGINE.md](docs/ENGINE.md)).

## Tests — run them before every PR

Forge has a real, executable test suite. It runs the **actual** `Bridge.lua`, `Store.lua`, and
`Protocol.lua` source under a faithful Roblox API mock (no Studio needed), plus validates the
asset payloads the engine produces against the MCP tool schemas.

```bash
cd tests
./run.sh
```

This regenerates the embedded source bundle, runs the protocol + asset suites under `luau`,
lints with selene, checks formatting, and builds the plugin. It must print
`ALL CHECKS PASSED ✓` and exit 0.

See [tests/README.md](tests/README.md) for how the harness works and how to add scenarios.

### What the tests cover

- **Protocol** (`tests/test_protocol.luau`): the full request lifecycle (enqueue → claim →
  stream → done/error), the commit barrier, atomic claim, the bridge-rebuild rewire, id
  persistence across restarts, timeout/reclaim, and style-profile round-trip.
- **Assets** (`tests/test_assets.luau`): real `create_build` and `create_ui_tree` payloads for
  real prompts, checked against the robloxstudio MCP schemas (valid classes/shapes/materials,
  part counts within limits, well-formed UDim2/Color3, JSON round-trip).

Tests are mutation-checked: the critical scenarios are written so that breaking the production
code makes them fail. If you change `Bridge.lua`/`Store.lua`/`Protocol.lua`, the suite runs the
new source automatically (the bundle is regenerated by `run.sh`).

## Code style

- Run `stylua src/` before committing — formatting is enforced.
- `selene src/` must report **0 errors / 0 parse errors**. The only acceptable warnings are the
  cosmetic `roblox_manual_fromscale_or_fromoffset` hints.
- Keep modules dependency-free (no Roact/Fusion) so the plugin stays install-free and portable.
- Field/enum names that cross the plugin↔engine boundary live in `src/Bridge/Protocol.lua` and
  are mirrored in `engine/forge-engine.md` + `docs/PROTOCOL.md` — change all three together.

## Pull requests

1. Branch off `main`.
2. Make your change; add or update a test scenario that proves it.
3. Run `tests/run.sh` — it must pass.
4. Open a PR describing the change and what you tested.

## Reporting bugs

Open an issue with: what you did, what you expected, what happened, and (if it's a generation
issue) the prompt and the result manifest from the chat panel.
